# Purpose

This document tracks all approved documentation work that has not yet been completed.

It exists to ensure documentation changes are never forgotten, lost between development sessions, or mixed into production engineering tasks.

This document is **not** a product backlog.

This document is **not** a sprint task list.

This document contains **documentation work only**.

---

# Engineering Rules

Only documentation tasks belong in this queue.

Examples:

✓ Create architecture documents

✓ Update engineering standards

✓ Update project documentation

✓ Create future architecture documents

✓ Update decision records

The following do **not** belong here:

✗ Code changes

✗ HTML changes

✗ CSS changes

✗ JavaScript changes

✗ Worker development

✗ Production testing

Those belong in the active sprint.

---

# Task Status

Documentation tasks may have one of the following status values:

- Waiting
- In Progress
- Review
- Complete
- Cancelled

Completed tasks are removed from the queue after verification.

New documentation work may only be added after a formal Decision Closeout.

- Title
- Purpose
- Priority
- Status

Tasks remain in this queue until completed or cancelled.
