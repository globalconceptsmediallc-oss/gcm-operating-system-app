# Purpose

This document defines the product vision for the Global Concepts Media Operating System (GCM OS).

The Architecture defines **how** the system works.

This document defines **what** the system is intended to become.

If this document conflicts with implementation details, the product vision defined here takes precedence. Engineering should adapt to support the product—not the other way around.

---

# Product Mission

GCM OS exists to help Global Concepts Media consistently:

1. Acquire better clients.
2. Deliver measurable business improvements.
3. Prove the value of completed work.
4. Turn successful client engagements into future revenue.

GCM OS is not a reporting platform.

GCM OS is the operating system used to run the agency.

---

# Product Philosophy

The consulting workflow is the product.

Artificial Intelligence is an enhancement layer.

The Business Record remains the single source of truth.

Every recommendation must ultimately become one of four things:

- A conversation
- A task
- A measurable improvement
- Proof of value

If intelligence does not improve the consulting workflow, it has not fulfilled its purpose.

# Core Product Principle

The intelligence engines are not the product.

The consulting workflow is the product.

The engines exist to make every business conversation more valuable.

---

# Product Rules

Every feature must support at least one of the following objectives:

- Acquire clients
- Deliver better work
- Prove measurable results
- Grow the agency

If a proposed feature does not directly support one of these objectives, it is deferred.

---

# Development Rules

One screen at a time.

One responsibility per screen.

Fresh-install files only.

Deploy.

Test.

Verify.

Lock.

Proceed.

Do not redesign previously approved architecture without verified business evidence.

Do not build intelligence that cannot be used within the consulting workflow.

Every completed sprint must produce something usable with real prospects or clients.

---

# MVP Success Criteria

Version 1.0 is considered operational when Andy can:

- Select prospects to contact.
- Review AI-powered client research.
- Conduct better discovery conversations.
- Deliver a paid 90-Day Growth Review.
- Build an implementation roadmap.
- Manage weekly client work.
- Measure completed improvements.
- Automatically generate Proof of Work.
- Automatically generate Case Studies.
- Repeat the process consistently across every client.

---

# Canonical Product Statement

**GCM OS is an Agency Operating System that combines AI-powered consulting intelligence with a structured consulting workflow to help Global Concepts Media acquire clients, deliver measurable improvements, prove results, and continuously grow through evidence-based consulting.**

---

# Product Decision Test

Before any new feature, engine, prompt, or workflow is approved, answer the following question:

**Does this make it easier to:**

- Acquire clients?
- Deliver work?
- Prove results?
- Grow the agency?

If the answer is **no**, the feature is deferred until a future release.

This rule protects the product from unnecessary complexity and ensures GCM OS remains focused on helping Global Concepts Media run and grow its business.
