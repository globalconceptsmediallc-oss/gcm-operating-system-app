# GCM OS CAPABILITIES

**Document Version:** 1.1

**Status:** Active

**Last Updated:** 2026-07-06

**Purpose:** Define every consulting capability within GCM OS and the
consulting question each capability answers.

------------------------------------------------------------------------

# Purpose

GCM OS is organized around consulting capabilities, not individual
evidence sources.

Each capability answers one consulting question, organizes observable
evidence, and contributes standardized information to the Business
Record.

The Business Record remains the single source of truth.

------------------------------------------------------------------------

# Core Principles

-   Every capability answers one consulting question.
-   Every capability contributes evidence to the Business Record.
-   Capabilities remain stable while evidence sources evolve.
-   Unknown information remains Unknown.
-   Observable evidence before assumptions.

------------------------------------------------------------------------

# Capability Evolution Rules

1.  Define the consulting question.
2.  Build Version 1 using the simplest reliable evidence source.
3.  Standardize capability output.
4.  Merge into the Business Record.
5.  Expand evidence sources without changing the capability's
    responsibility.

------------------------------------------------------------------------

# Long-Term Objective

Build a library of modular consulting capabilities that continuously
enrich the Business Record and prepare consultants for better first
conversations.
