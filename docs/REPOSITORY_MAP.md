# Purpose

This document defines the responsibilities of every repository within the Global Concepts Media ecosystem.

Every new feature, document, page, engine, or standard must belong to exactly one repository.

If there is uncertainty about where something belongs, this document is the source of truth.

---

# Repository Principles

Each repository has one responsibility.

Repositories communicate with each other.

Repositories do not duplicate responsibilities.

Marketing belongs in the website.

Consulting belongs in GCM OS.

Standards belong in the standards repositories.

# Repository Responsibilities

## globalconceptsmedia

Purpose

Public marketing website.

Responsibilities

- Homepage
- Service Pages
- SEO
- Google Ads
- Industries
- Case Studies
- Contact
- Privacy
- Branding
- Lead Generation

Does NOT contain

- AI
- Workers
- Business Snapshot logic
- Dashboard
- Business Record
- Consulting Engines

Primary Goal

Convert visitors into GCM OS users.

---

## gcm-operating-system-app

Purpose

The AI-powered Consulting Operating System.

Responsibilities

- Business Snapshot
- Processing
- Snapshot Results
- Worker
- Engine Hub
- Capability Engines
- Business Record
- Consulting Intelligence
- Dashboard
- Future Client Portal

Does NOT contain

- Marketing pages
- SEO landing pages
- Case Studies
- General website content

Primary Goal

Transform observable business evidence into consulting intelligence.

---

## gcm-business-growth-system

Purpose

Responsibilities

- Metrics
- Operating Philosophy
- Consulting Methodology

Primary Goal

Define how Global Concepts Media improves businesses.

## gcm-client-acquisition-system

# Engineering Rules

Every page has one owner.

Every feature has one repository.

Never duplicate functionality between repositories.

Never duplicate documentation.

When in doubt:

Business Standard?
→ Business Growth System

Client Acquisition?
→ Client Acquisition System

Marketing?
→ Website

Consulting Intelligence?
→ GCM OS
